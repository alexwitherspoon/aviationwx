Runway display mockup - hybrid (Option E)

Refining hybrid runways in Airport Information. Calm wind tags appear on designated ends only when wind is calm (< 3 kt, FAA METAR standard); at 3 kt and above, runway choice is the PIC's decision. Mockup values use ft and kts for readability; production must follow dashboard unit preferences (see notes).

Implementation notes
  • Runway dimensions - Mock shows feet (e.g. 5100 FT × 75 FT). When shipped, length and width must convert with the dashboard distance unit toggle (ft / m), same as field elevation, density altitude, and ceiling.
  • Per-end wind components - Mock suffixes use kt. When shipped, headwind and crosswind values must use formatWindSpeed() and the active wind unit label (kts, mph, km/h) from the Runway Wind toggle, consistent with wind speed, gusts, and gust factor.
  • Calm wind threshold - Use CALM_WIND_THRESHOLD (3 kt), matching FAA METAR calm encoding (00000KT) and dashboard "Calm" display. Do not use the AIM 5 kt calm-wind-runway assignment threshold.
  • Calm wind runway tags - Show only while wind is < 3 kt. At 3 kt and above, hide tags; runway selection is the PIC's decision (ATC, performance, traffic pattern, etc.). Per-end wind components remain visible at all speeds.
  • Unit conversion - Calm detection stays in knots internally; only displayed numbers and labels change with the user's unit choice.
